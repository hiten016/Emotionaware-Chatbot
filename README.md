# Emotionaware-API-for-LLMs

This project builds a plug-and-play API layer for LLMs that improves responses by adding emotion awareness, intent detection, safety handling, and user feedback learning. It acts as a middleware system on top of any LLM, enabling developers to generate more empathetic, safe, and context-aware responses without modifying the base model. The system first analyzes user input to detect emotion (multi-label) and intent (question, rant, help-seeking). Based on this, it dynamically routes and conditions prompts to guide the LLM toward appropriate responses (empathetic tone, safe refusal, etc.). A built-in feedback loop collects user ratings and converts them into training data to improve the model using Supervised Fine-Tuning (SFT) and Direct Preference Optimization (DPO). The API is deployed using a real-time inference stack, supporting low-latency responses, session memory, and continuous improvement.
